# UI Mockup Reference Manifest

This file preserves the current visual source-of-truth references that were supplied in the working chat thread for the 2026-05-17 UI modernization pass.

## Purpose

- Keep the mockup intent available in-repo even if chat context is compacted later.
- Give the next UI polish pass a stable checklist of the screenshots and compositions we are targeting.
- Support the next planned art-pack build for Alliance-first fidelity.

## Current Reference Set

### 1. Style Guide Board

This is the large style-guide composition board that defines:

- crafting-quality tier visuals
- semantic status colors
- theme previews
- dashboard layout target
- inventory, history, minimums, request admin, exports, about, options, and request-only mockups
- the overall Blizzard-inspired dark fantasy visual language

Use this board as the broadest visual contract for:

- panel framing
- spacing rhythm
- typography hierarchy
- icon density
- button treatment
- table density
- surface layering

### 2. Alliance Dashboard Mockup

This is the strongest single-screen fidelity target right now and should drive the first art-pack-assisted pass.

Key traits to match:

- deep blue-black matte background surfaces
- thin gold trim and separators
- heraldic sidebar crest treatment
- left nav with icon-led rows and a stronger active plate
- metric cards with distinct colored surface variants
- structured `Top 5 Most Used` rows with item icons, quality, bars, and values
- structured `Recent Activity` list with badges, actor, and time
- quick-action cards with large icons and short descriptions
- bottom legend and status treatment

### 3. Request-Only Wizard Mockup

This is the target for the compact member-facing request flow.

Key traits to preserve or continue toward:

- step-by-step wizard framing
- large numbered steps
- compact dark panels
- stronger primary CTA treatment
- visual review step with clear hierarchy
- friendly completion state

## Current Reality

- The reusable UI scaffolding is committed and pushed.
- The addon does **not** yet visually match these references closely enough.
- A later dashboard composition pass was rolled back one iteration because the live result overextended the layout before the surface art was good enough.
- The next meaningful visual milestone is building an addon-local art pack so these references can be matched more literally.
